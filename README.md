## 项目简介

### 项目背景

OilGuard是一个基于**深度学习**与**大语言模型**的抽油机井工况智能诊断系统。该系统集成了示功图生成、数据预处理、模型训练、工况诊断、性能评估和大模型智能助手等完整功能模块，为油田生产提供智能化的故障诊断解决方案。

### 核心能力

- **12种工况识别**：支持识别工作正常、供液不足、气体影响、气锁、上碰泵、下碰泵、游动阀关闭迟缓、柱塞脱出泵工作筒、游动阀漏、固定阀漏、砂影响+供液不足、惯性影响+工作正常等12种抽油机井工况状态

- **多模型支持**：集成RepConvNeSt、ConvNext、ResNet、EfficientNet等20+种先进的深度学习模型

- **大模型智能助手**：基于阿里云通义千问（Qwen）大模型，提供智能问答、模块介绍、故障分析等交互式服务

- **工业控制系统网络安全诊断**：集成机器学习模型进行工控网络流量异常检测，保障数据安全

## 🚀 快速开始

### 环境配置

```bash
# 克隆项目
git clone <repository_url>
cd OilGuard  # 原文件夹名

# 创建虚拟环境（推荐）
conda create -n oil_diagnosis python=3.8
conda activate oil_diagnosis

# 安装依赖
pip install -r requirements.txt
```

### 配置API密钥

```bash
# 配置 wandb（用于训练监控）
wandb login
export WANDB_API_KEY=your_wandb_api_key

# 配置阿里云通义千问（用于智能助手）
export DASHSCOPE_API_KEY=your_dashscope_api_key
```

### 启动Web服务

```bash
# 启动Gradio服务
python run.py

# 服务启动后，访问本地地址
# http://127.0.0.1:7860
```

### 命令行训练

```bash
# 使用RepConvNeSt模型训练
python main.py --model_name repconvnest --epoch 100 --batch_size 64

# 使用ResNet50模型训练
python main.py --model_name resnet50 --epoch 100 --batch_size 64

# 使用知识蒸馏
python main.py --model_name student_model --kd --teacher_path path/to/teacher
```

### 核心设计组件

#### 1. 重参数化非对称卷积块 (ACBlock)

ACBlock将标准的k×k卷积分解为三个卷积分支的并行组合，在推理阶段可融合为单一卷积核：

```
训练阶段 (多分支):                              推理阶段 (单分支):

┌──────────────────────────────────────┐     ┌──────────────────────────────────────┐
│                                      │     │                                      │
│  ┌──────────┐   ┌──────────┐         │     │                                      │
│  │ Square   │   │  Hor     │         │     │                                      │
│  │ Conv k×k │ + │ Conv 1×k │ ─┐      │     │     Conv k×k (融合权重)              │
│  └──────────┘   └──────────┘  │      │     │     等效于三个分支的和                │
│       │              │       │      │     │                                      │
│       ▼              ▼       ▼      │     │                                      │
│  ┌──────────┐   ┌──────────┐ ┌──────┴──┐  │     └──────────────────────────────────────┘
│  │ Square   │ + │  Hor     │ │  Ver    │  │
│  │   BN     │   │   BN     │ │   BN    │  │
│  └────┬─────┘   └────┬─────┘ └───┬────┘  │
│       └───────────────┴──────────┘        │
│                    │                       │
│                    ▼                       │
│              输出: F(x)                     │
└──────────────────────────────────────┘
```

**数学表达**：
- 训练时：$F(x) = BN_{square}(Conv_{k×k}(x)) + BN_{hor}(Conv_{1×k}(x)) + BN_{ver}(Conv_{k×1}(x))$
- 推理时：$F(x) = Conv_{fused}(x)$，其中 $W_{fused} = W_{k×k} \oplus W_{1×k} \oplus W_{k×1}$

**优势**：
- 增加卷积核的多样性，提取更丰富的特征
- 推理时零成本，部署效率高
- 保持模型表达能力的同时减少参数量

#### 2. 多维度并行重参数化注意力 (MPRAttention)

MPRAttention创新性地融合了三个维度的注意力机制：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        MPRAttention 架构                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│                          输入特征图 x (B, C, H, W)                       │
│                                      │                                   │
│          ┌───────────────────────────┼───────────────────────────┐      │
│          │                           │                           │      │
│          ▼                           ▼                           ▼      │
│  ┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐  │
│  │   通道注意力     │     │   空间注意力     │     │   SimAM注意力   │  │
│  │   (ECA)         │     │   (ACBlock)     │     │   (3D)         │  │
│  │                 │     │                 │     │                 │  │
│  │ AvgPool → 1D    │     │ MaxPool+AvgPool │     │ 像素级权重计算  │  │
│  │ Conv1D → Sigmoid │     │ → ACBlock → σ   │     │ 基于方差归一化  │  │
│  │                 │     │                 │     │                 │  │
│  │ F_channel =     │     │ F_spatial =     │     │ F_simam =      │  │
│  │ x ⊙ Attention   │     │ x ⊙ Attention   │     │ x ⊙ sigmoid(y) │  │
│  └────────┬────────┘     └────────┬────────┘     └────────┬────────┘  │
│           │                      │                      │            │
│           └──────────────────────┼──────────────────────┘            │
│                                  ▼                                     │
│                         输出 = x + (F_channel + F_spatial + F_simam) / 3 │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**各注意力机制详解**：

| 注意力类型 | 原理 | 作用 |
|-----------|------|------|
| **ECA (通道注意力)** | 使用1D卷积自适应学习通道间的相关性 | 增强重要通道，抑制无关通道 |
| **ACBlock (空间注意力)** | 融合最大池化与平均池化，通过非对称卷积学习空间权重 | 聚焦示功图中的关键区域 |
| **SimAM (3D注意力)** | 基于像素方差计算能量函数，无需额外参数 | 增强显著性区域的特征表达 |

#### 3. 倒置残差块 (InvertedResidual)

RepConvNeSt采用两种模式的倒置残差块：

**Mode 1 - 轻量残差模式（Stage 4）**：
```
输入 ──► Depthwise Conv 3×3 ──► PWConv1 ──► ReLU ──► GRN ──► PWConv2 ──► DropPath ──┐
                                                                                          │
                                                                                          ▼
                                                                                      输出 = 输入 + 残差
```
- 标准倒置残差结构
- 直接相加（add）而非通道拼接
- 适用于深层特征提取

**Mode 2 - 深度混洗模式（Stage 1-3）**：
```
输入 ──► [分支1: 空洞深度卷积 → 1×1卷积]
   │
   ├─► 通道分割 (chunks=2)
   │
   └─► [分支2: 深度可分离卷积 → PWConv1 → ReLU → GRN → PWConv2 → DropPath]
                                       │
                                       ▼
                                  通道混洗 (shuffle=2)
                                       │
                                       ▼
                                  输出 = Cat([分支1, 分支2])
```
- 分支1使用空洞卷积(dilation=2)扩大感受野
- 分支2使用门控线性单元(GLU)增强非线性
- 通道混洗促进跨组信息交流

#### 4. 全局响应归一化 (GRN)

GRN是一种通道级别的归一化方法，用于增强通道间的竞争：

```python
Gx = ||x||₂  # 计算通道的L2范数
Nx = Gx / (Gx.mean(dim=-1) + ε)  # 归一化
output = γ * (x * Nx) + β + x     # 仿射变换 + 残差
```

**与LayerNorm/GroupNorm的区别**：
- GRN仅在通道维度进行归一化，保留空间维度完整性
- 无需显式分组，自动学习通道间的依赖关系
- 计算效率高，轻量级

#### 5. 余弦分类器 (Cos_Classifier)

传统全连接分类器使用 $y = Wx + b$，RepConvNeSt采用余弦分类器：

$$y_i = s \cdot \frac{w_i \cdot x}{||w_i|| \cdot ||x||} + b_i = s \cdot \cos(\theta_i) + b_i$$

**设计优势**：
- **特征归一化**：消除特征尺度的影响，增强模型稳定性
- **权重归一化**：使分类边界为超平面，决策更清晰
- **可学习缩放因子(s)**：动态调整 logits 分布，优化softmax效果
- **更适合度量学习**：余弦相似度天然适合分类任务

## 🔧 工况类别说明

系统支持识别以下**12种**抽油机井工况：

| 序号 | 工况名称 | 英文名 | 说明 |
|-----|---------|--------|------|
| 1 | 工作正常 | Normal Operation | 抽油机正常运行 |
| 2 | 供液不足 | Insufficient Fluid Supply | 油井供液能力不足 |
| 3 | 气体影响 | Gas Interference | 气体影响泵效 |
| 4 | 气锁 | Gas Lock | 气体阻塞泵腔 |
| 5 | 上碰泵 | Top Collision | 柱塞撞击上死点 |
| 6 | 下碰泵 | Bottom Collision | 柱塞撞击下死点 |
| 7 | 游动阀关闭迟缓 | Traveling Valve Slow Closing | 游动阀响应延迟 |
| 8 | 柱塞脱出泵工作筒 | Plunger Out of Barrel | 柱塞异常运行 |
| 9 | 游动阀漏 | Traveling Valve Leak | 游动阀密封失效 |
| 10 | 固定阀漏 | Standing Valve Leak | 固定阀密封失效 |
| 11 | 砂影响+供液不足 | Sand Influence + Insufficient | 多重因素影响 |
| 12 | 惯性影响+工作正常 | Inertia Effect + Normal | 惯性因素+正常 |

## 📊 评估指标说明

### 分类性能指标

| 指标 | 计算公式 | 解释 |
|-----|---------|------|
| Accuracy | (TP+TN)/(TP+TN+FP+FN) | 总体正确率 |
| Precision | TP/(TP+FP) | 查准率 |
| Recall | TP/(TP+FN) | 查全率/召回率 |
| F1-Score | 2×Precision×Recall/(P+R) | 调和平均值 |
| AUC | ROC曲线下面积 | 分类器性能 |
| Kappa | (Po-Pe)/(1-Pe) | 一致性检验 |

### 可视化图表

- **混淆矩阵**：展示各类别的预测与真实分布
- **ROC曲线**：多分类ROC曲线，每个类别一条
- **PR曲线**：多分类PR曲线，侧重正类分析
- **t-SNE图**：高维特征降维可视化

## 📝 常见问题

### Q1: 训练中断后如何恢复？

```bash
python main.py --resume --save_path path/to/your/model
```

### Q2: 如何使用GPU训练？

```bash
# CUDA设备自动检测
python main.py --device 0

# 指定多GPU
python main.py --device 0,1,2,3
```

### Q3: wandb显示不了训练曲线？

1. 检查网络连接
2. 确认WANDB_API_KEY正确配置
3. 尝试 `wandb sync` 同步数据

### Q4: 通义千问API调用失败？

1. 确认DASHSCOPE_API_KEY有效
2. 检查API配额是否用尽
3. 网络是否能访问阿里云服务

### Q5: 如何导出训练好的模型？

```bash
# 导出为TorchScript
python export.py --save_path path/to/model --model_type torchscript

# 导出为ONNX
python export.py --save_path path/to/model --model_type onnx
```
